  [1X4.1 [33X[0;0YIntroduction[133X[101X
  
  [33X[0;0YStrips  are  the  principal  objects  of  the  [5XSBStrips[105X  package.  They  are
  syzygy-minded representations of string graphs.[133X
  
  
  [1X4.2 [33X[0;0YConstructing strips[133X[101X
  
  
  [1X4.2-1 [33X[0;0YStripify[133X[101X
  
  [33X[1;0Y[29X[2XStripify[102X( [3Xarr[103X, [3XN[103X, [3Xint_list[103X ) [32X method[133X
  [33X[1;0Y[29X[2XStripify[102X( [3Xpath[103X ) [32X method[133X
  
  [33X[0;0YArguments (first method): [3Xarr[103X, an arrow in a SB algebra (see note below); [3XN[103X,
  an  integer  which  is  either [10X1[110X or [10X-1[110X; [3Xint_list[103X, a (possibly empty) list of
  nonzero integers whose entries are alternately positive and negative).[133X
  
  [33X[0;0YArgument (second method): [3Xpath[103X, a path in a SB algebra.[133X
  
  [33X[0;0Y([13XNote.[113X  Remember  that  vertices and arrows in a SB algebra, which is to say
  the  elements in the algebra corresponding to the vertices and arrows of the
  quiver, can be easily accessed using [2X.[102X ([14XQPA: . for a path algebra[114X), and that
  these  can  be multiplied together using [2X\*[102X ([14XReference: *[114X) to make a path in
  the SB algebra.)[133X
  
  [6XReturns:[106X  [33X[0;10Ythe strip specified by these data[133X
  
  [33X[0;0YThe first method is intended for specifying arbitrary string( graphs) over a
  SB algebra to [5XGAP[105X. The second method is more specialized, being intended for
  specifying  those  string(  graph)s  where  all  arrows  point  in  the same
  direction.  This  includes  the vacuous case where the string (graph) has no
  arrows.[133X
  
  [33X[0;0YFor  the  first  method, suppose you draw your string graph on the page as a
  linear  graph  with  some  arrows  pointing  to  the  right  (the "positive"
  direction)  and  some  to  the  left (the "negative" direction). See further
  below for examples.[133X
  
  [33X[0;0YThe  first  arrow  (ie,  the  leftmost  one drawn on the page) is [3Xarr[103X. If it
  points  to  the  right (the "positive" direction), then set [3XN[103X to be [10X1[110X. If it
  points to the left (the "negative" direction), then set [3XN[103X to be [10X-1[110X.[133X
  
  [33X[0;0YNow,  ignore  that  first arrow [3Xarr[103X and look at the rest of the graph. It is
  made up of several paths that alternately point rightward and leftward. Each
  path  has  a  [13Xlength[113X;  that  is, the total number of arrows in it. Enter the
  lengths  of  these  paths  to  [3Xint_list[103X  in  the  order you read them, using
  positive  numbers  for  paths  pointing  rightwards and negative numbers for
  paths pointing leftwards.[133X
  
  [33X[0;0Y[5XSBStrips[105X  will  check that your data validily specify a strip. If it doesn't
  think they do, then it will throw up an Error message.[133X
  
  [33X[0;0YFor  the  second method, [5XSBStrips[105X directly infers the string (graph) and the
  SB algebra directly from [3Xpath[103X.[133X
