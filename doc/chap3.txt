  [33X[1;0Y[29X[2XStripify[102X( [3Xarr[103X, [3XN[103X, [3Xint_list[103X ) [32X method[133X
  
  [33X[0;0YArguments:  [3Xarr[103X,  the residue of an arrow in a special biserial algebra (see
  below);  [3XN[103X, an integer which is either [10X1[110X or [10X-1[110X; [3Xint_list[103X, a (possibly empty)
  list  of  nonzero  integers  whose  entries  are  alternately  positive  and
  negative).[133X
  
  [33X[0;0Y(Remember  that  residues  of  arrows  in  an  quiver  algebra can be easily
  accessed  using  the  [10X\.[110X  operation.  See  [2X.[102X ([14XQPA: . for a path algebra[114X) for
  details and see below for examples.)[133X
  
  [6XReturns:[106X  [33X[0;10Ythe strip specified by this data[133X
  
  [33X[0;0YRecall  that  [5XSBStrips[105X uses strip objects to represent the kind of decorated
  graph  that  representation  theorists call "strings". Now, suppose you draw
  that  string  on the page as a linear graph with some arrows pointing to the
  right  (the  "positive"  direction)  and  some  to  the left (the "negative"
  direction). See further below for examples.[133X
  
  [33X[0;0Y(Of course, this method assumes that the string contains at least one arrow.
  There  is  a  different, easier, method for strings comprising only a single
  vertex. Namely [2XStripify[102X called with the residue of a vertex.)[133X
  
  [33X[0;0YThe  first  arrow  (ie,  the  leftmost  one drawn on the page) is [3Xarr[103X. If it
  points  to  the  right (the "positive" direction), then set [3XN[103X to be [10X1[110X. If it
  points to the left (the "negative" direction), then set [3XN[103X to be [10X-1[110X.[133X
  
  [33X[0;0YNow,  ignore  that  first arrow [3Xarr[103X and look at the rest of the graph. It is
  made up of several paths that alternately point rightward and leftward. Each
  path  has  a  [13Xlength[113X;  that  is, the total number of arrows in it. Enter the
  lengths  of  these  paths  to  [3Xint_list[103X  in  the  order you read them, using
  positive  numbers  for  paths  pointing  rightwards and negative numbers for
  paths pointing leftwards.[133X
  
  [33X[0;0Y[5XSBStrips[105X  will  check that your data validily specify a strip. If it doesn't
  think they do, then it will throw up an Error message.[133X
  
  [33X[1;0Y[29X[2XStripify[102X( [3Xpath[103X ) [32X method[133X
  
  [33X[0;0YArguments: [3Xpath[103X, the residue (in a special biserial algebra) of some path.[133X
  
  [33X[0;0Y(Remember  that residues of vertices and arrows can be easily accessed using
  [2X.[102X  ([14XQPA:  .  for  a path algebra[114X), and that these can be multiplied together
  using [2X\*[102X ([14XReference: *[114X) to make a path.)[133X
  
  [6XReturns:[106X  [33X[0;10YThe strip corresponding to [3Xpath[103X[133X
  
  [33X[0;0YRecall that uniserial modules are string modules. The uniserial modules of a
  SB   algebra  are  in  [22X1[122X-to-[22X1[122X  correspondence  with  the  paths  [22Xp[122X  linearly
  independent  from  all  other paths. Therefore, this path is all you need to
  specify the strip.[133X
